# Tags

> Prefix: `/api/v1/tags`  
> Все ендпоинты требуют Bearer токен.  
> Теги принадлежат пользователю — он видит только свои.

---

## Объект тега

```json
{
  "id": "uuid",
  "name": "идея",
  "color": "#ff0000",
  "created_at": "..."
}
```

---

## GET `/tags`

Все теги пользователя.

**Response 200**
```json
{
  "items": [
    { "id": "uuid", "name": "идея", "color": "#ff0000", "created_at": "..." },
    { "id": "uuid", "name": "важно", "color": "#ff6b00", "created_at": "..." }
  ]
}
```

> Список без пагинации — у пользователя не предполагается тысяч тегов.

---

## POST `/tags`

Создать тег.

**Body**
```json
{ "name": "идея", "color": "#ff0000" }
```

**Валидация**
- `name` — обязательное, 1–100 символов
- `color` — HEX-цвет, например `#ff0000` или `#f00`

**Логика:** проверить уникальность `(owner_user_id, name)`.

**Response 201** — созданный тег.  
**Ошибки:** `409 already_exists` если тег с таким именем уже есть.

---

## PATCH `/tags/{tag_id}`

Обновить тег.

**Body** (все опциональны)
```json
{ "name": "новое имя", "color": "#00ff00" }
```

**Логика:** проверить уникальность нового имени если оно передано.

**Response 200** — обновлённый тег.

---

## DELETE `/tags/{tag_id}`

Удалить тег.

**Логика**
1. Удалить все записи в `note_tags` где `tag_id = id`
2. Удалить тег из `tags`

**Response 204**
