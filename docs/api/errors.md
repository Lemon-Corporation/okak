# Ошибки

## Формат ответа

Все ошибки возвращают JSON:

```json
{
  "detail": "Человекочитаемое описание",
  "code": "machine_readable_code"
}
```

---

## HTTP-коды

| Код | Когда |
|-----|-------|
| `400` | Невалидные данные в запросе |
| `401` | Нет токена или токен протух |
| `403` | Токен валидный, но нет доступа к ресурсу |
| `404` | Ресурс не найден |
| `409` | Конфликт (дубликат) |
| `422` | Pydantic не смог распарсить body (автоматически от FastAPI) |
| `500` | Внутренняя ошибка сервера |

---

## Машиночитаемые коды

| code | Статус | Описание |
|------|--------|---------|
| `email_taken` | 409 | Email уже зарегистрирован |
| `invalid_credentials` | 401 | Неверный email или пароль |
| `token_expired` | 401 | JWT истёк |
| `token_invalid` | 401 | JWT не валидный |
| `not_found` | 404 | Ресурс не найден |
| `forbidden` | 403 | Ресурс принадлежит другому пользователю |
| `already_exists` | 409 | Связь уже существует (file, tag) |
| `file_too_large` | 400 | Файл превышает лимит размера |
| `invalid_mime_type` | 400 | Тип файла не разрешён |
| `file_not_on_disk` | 500 | Файл есть в БД, но не найден на диске |
| `query_too_short` | 400 | Поисковый запрос менее 2 символов |

---

## Примеры

**401 — нет токена**
```json
{ "detail": "Not authenticated", "code": "token_invalid" }
```

**403 — чужой ресурс**
```json
{ "detail": "Access denied", "code": "forbidden" }
```

**409 — тег уже прикреплён**
```json
{ "detail": "Tag already attached to this note", "code": "already_exists" }
```

**400 — файл слишком большой**
```json
{ "detail": "File exceeds 50 MB limit", "code": "file_too_large" }
```
